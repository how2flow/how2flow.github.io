플랫폼 디바이스 드라이버 리뷰를 [하나](/posts/gpiomem) 했습니다.<br>
이번엔 특정 아키텍처가 아닌 Linux에서 단일 모듈로 올라가는 케이스를 리뷰합니다.<br>
저는 I/O쪽으로 개발을 주로 하다보니, 관련 드라이버가 좋을것 같았습니다.<br>
그 중에서 많이 사용하는 인터페이스 i2c-dev 드라이버를 선택했습니다.<br>

## 개요

**i2c-dev** 드라이버는 i2c 하드웨어와 연결해주는 드라이버 입니다. <br>
앞서 gpiomem과 다르게 i2c-dev는 generic한 모듈인데요, 그차이는 <br>
i2c-dev는 브릿지 역할을 하는 커널 모듈이고, gpiomem은 각 아키텍처의 메모리 구조에 따라<br>
설계가 달라지기 때문입니다.<br>

그래서 모델을 콕 집어 선택하는 것이 아닙니다.<br>

## 코드 리뷰

코드 리뷰는 커널 5.10.y 기준 입니다.<br>

<span style="{{ site.code }}">\<linux/kernel.h\></span> : 커널 API 지원 <br>
<span style="{{ site.code }}">\<linux/list.h\></span> : 커널에서 사용되는 연결리스트 지원<br>
<span style="{{ site.code }}">\<linux/module.h\></span> : 커널 모듈과 관련 매크로 지원<br>
<span style="{{ site.code }}">\<linux/notifier.h\></span> : 커널에서 발생하는 이벤트와 함수포인터를 매칭<br>
<span style="{{ site.code }}">\<linux/slab.h\></span> : <span style="{{ site.code }}">kzalloc</span> , <span style="{{ site.code }}">kfree</span> 등 커널 메모리 작업 지원<br>
<span style="{{ site.code }}">\<linux/uaccess.h\></span> : 커널에서 유저 스페이스와 커널 스페이스 간의 안전한 메모리 접근 지원<br>

```
static int i2cdev_open(struct inode *inode, struct file *file)
{
	unsigned int minor = iminor(inode);
	struct i2c_client *client;
	struct i2c_adapter *adap;
	
	adap = i2c_get_adapter(minor);
	if (!adap) 
		return -ENODEV;
	
	/* This creates an anonymous i2c_client, which may later be
	 * pointed to some address using I2C_SLAVE or I2C_SLAVE_FORCE.
	 *
	 * This client is ** NEVER REGISTERED ** with the driver model
	 * or I2C core code!!  It just holds private copies of addressing
	 * information and maybe a PEC flag.
	 */
	client = kzalloc(sizeof(*client), GFP_KERNEL);
	if (!client) {
		i2c_put_adapter(adap);
		return -ENOMEM;
	}
	snprintf(client->name, I2C_NAME_SIZE, "i2c-dev %d", adap->nr);
	
	client->adapter = adap;
	file->private_data = client;
	
	return 0;
}

static int i2cdev_release(struct inode *inode, struct file *file)
{
	struct i2c_client *client = file->private_data;
	
	i2c_put_adapter(client->adapter);
	kfree(client);
	file->private_data = NULL;
	
	return 0;
}
```

<span style="{{ site.code }}">file_operations</span> 쪽 함수의 내용이 너무 많아서 이 포스팅에서는 자세히 다루지 않습니다.<br>

<span style="{{ site.code }}">.llseek</span> : 파일 위치를 이동시키는 파일 포인터 입니다. <span style="{{ site.code }}">no_llseek</span> 이므로 사용하지 않습니다.<br>
<span style="{{ site.code }}">.read</span> , <span style="{{ site.code }}">.write</span> 는 각각 드라이버 읽기/쓰기 동작 입니다.<br>

<span style="{{ site.code }}">.compat_ioctl</span> : 32비트 장치의 호환을 위해 존재합니다.<br>

```
static int __init i2c_dev_init(void)
{
	int res;
	
	printk(KERN_INFO "i2c /dev entries driver\n");
	
	res = register_chrdev_region(MKDEV(I2C_MAJOR, 0), I2C_MINORS, "i2c");
	if (res)
		goto out;
	
	i2c_dev_class = class_create(THIS_MODULE, "i2c-dev");
	if (IS_ERR(i2c_dev_class)) {
		res = PTR_ERR(i2c_dev_class);
		goto out_unreg_chrdev;
	}
	i2c_dev_class->dev_groups = i2c_groups;
	
	/* Keep track of adapters which will be added or removed later */
	res = bus_register_notifier(&i2c_bus_type, &i2cdev_notifier);
	if (res)
		goto out_unreg_class;
	
	/* Bind to already existing adapters right away */
	i2c_for_each_dev(NULL, i2cdev_attach_adapter);
	
	return 0;
	
	out_unreg_class:
		class_destroy(i2c_dev_class);
	out_unreg_chrdev:
		unregister_chrdev_region(MKDEV(I2C_MAJOR, 0), I2C_MINORS);
	out:
		printk(KERN_ERR "%s: Driver Initialisation failed\n", __FILE__);
		return res;
}

static void __exit i2c_dev_exit(void)
{
	bus_unregister_notifier(&i2c_bus_type, &i2cdev_notifier);
	i2c_for_each_dev(NULL, i2cdev_detach_adapter);
	class_destroy(i2c_dev_class);
	unregister_chrdev_region(MKDEV(I2C_MAJOR, 0), I2C_MINORS);
}
```

<span style="{{ site.code }}">static int \__init i2c_dev_init</span> 부터 보겠습니다.<br>
우선, \__init 은 커널에서 초기화 과정에서만 호출된다는 의미입니다.<br>
그리고 <span style="{{ site.code }}">register_chrdev_region</span> 함수는 드라이버의 major, minor 번호를 부여합니다.<br>

<span style="{{ site.code }}">i2c_dev_class = class_create(THIS_MODULE, "i2c-dev");</span> 라인은<br>
sysfs아래에 노드를 만드는 코드입니다. (goto 에러 핸들링은 리뷰에서 제외합니다.)<br>

<span style="{{ site.code }}">i2c_dev_class->dev_groups = i2c_groups;</span> 은 디바이스의 그룹을 지정해 줍니다.<br>
그룹은 <span style="{{ site.code }}">drivers/i2c/i2c-core-base.c</span> 에 정의 되어 있습니다.
```
truct device_type i2c_adapter_type = {
	.groups     = i2c_adapter_groups,
	.release    = i2c_adapter_dev_release,
};
EXPORT_SYMBOL_GPL(i2c_adapter_type);
```

이 코드를 가져온 이유는, 매크로 하나를 소개하려고 합니다.<br>
<span style="{{ site.code }}">EXPORT_SYMBOL_GPL</span> 은 함수를 커널의 심볼테이블에 등록하는 역할을 합니다.<br>
각 커널의 소스들은 보안을 위해 다른 경로의 소스의 함수나 변수를 참조하지 못합니다.<br>
커널은 이 요소들을 각 심볼로써 관리를 합니다.<br>
심볼로 export 해야 다른 쪽에서 가져다 사용할 수 있습니다.<br>
일반적인 c 프로그램들의 extern 처럼 외부 참조 할 수 있다, 생각하면 됩니다.<br>

다시 소스코드로 돌아와서,
```
	/* Keep track of adapters which will be added or removed later */
	res = bus_register_notifier(&i2c_bus_type, &i2cdev_notifier);
	if (res)
		goto out_unreg_class;
```

i2cdev의 <span style="{{ site.code }}">notifier</span> 를 등록하는 함수입니다.<br>
<span style="{{ site.code }}">notifier</span> 는 커널에서 발생하는 이벤트를 감지하고 이를 커널이나 사용자에게 알리는 역할을 합니다.<br>

<span style="{{ site.code }}">i2cdev_notifier</span> 에 대한 내용입니다.
```
static int i2cdev_notifier_call(struct notifier_block *nb, unsigned long action,
			void *data)
{
	struct device *dev = data;
	
	switch (action) {
	case BUS_NOTIFY_ADD_DEVICE:
		return i2cdev_attach_adapter(dev, NULL);
	case BUS_NOTIFY_DEL_DEVICE:
		return i2cdev_detach_adapter(dev, NULL);
	}
	
	return 0;
}

static struct notifier_block i2cdev_notifier = {
    .notifier_call = i2cdev_notifier_call,
};
```
switch 문으로 action(이벤트)에 따라 호출할 함수가 정의되어 있습니다.<br>
i2c 장치가 추가 되거나 제거 될 때 <span style="{{ site.code }}">i2cdev_attach_adapter</span> / <span style="{{ site.code }}">i2cdev_detach_adapter</span> 함수를 호출 합니다.<br>

<details>
<summary><B>+ Notifier Chain</B></summary>
  <p>
    <br> 
    <B>bus_register_notifier</B> 함수가 하는 일은 디바이스 버스를 <span style="{{ site.code }}">Notifier Chain</span> 에 등록하는 함수입니다.<br>
    <span style="{{ site.code }}">Notifier Chain</span> 는 이벤트 발생 시에 여러 컴포넌트에 알리기 위한 매커니즘입니다.<br>
    이벤트를 발생시키는 컴포넌트는 체인에 등록된 다른 컴포넌트들에게 알림을 보낼 수 있습니다.<br>
    전반적인 리눅스 커널 시스템이 이벤트에 대해 효과적으로 반응하고 처리할 수 있습니다.<br>
    
    함수를 파다 보면 다음과 같은 코드를 볼 수 있습니다.
    <pre>
    static int notifier_chain_register(struct notifier_block **nl,
    				struct notifier_block *n)
    {
        while ((*nl) != NULL) {
            if (unlikely((*nl) == n)) {
                WARN(1, "double register detected");
                return 0;
            }
            if (n->priority > (*nl)->priority)
                break;
            nl = &((*nl)->next);
        }
        n->next = *nl;
        rcu_assign_pointer(*nl, n);
        return 0;
    }
    </pre>

    내용을 보면 Notifier Chain에 새로운 컴포넌트를 우선순위에 따라 추가하고 정렬합니다.<br><br>

  <details>
  <summary><B>++ Notifier 추가/정렬에 대하여..</B></summary>
    <p>
      <br>
      <B>while ((*nl) != NULL) {</B> 문 안에서 리스트를 순회합니다.<br>
      <B>if (n->priority > (*nl)->priority)</B> 에서 우선순위를 확인해서 우선순위가 더 높으면 순회를 끝내고 빠져나옵니다.<br>
      그렇지 않다면, <B>nl = &((*nl)->next);</B> 코드로 리스트를 다음으로 이동합니다.<br>
      루프문에서 빠져 나왔다면 그 위치에서 <B>n->next = *nl;</B> 현재의 block을 삽입합니다.<br>
      <B>rcu_assign_pointer(*nl, n);</B> <span style="{{ site.code }}">RCU</span> 매커니즘으로 포인터를 업데이트 합니다.<br><br>
      <code>#define rcu_assign_pointer(p, v)    do { (p) = (v); } while (0)</code><br>
      <span style="{{ site.code }}">RCU</span> 에 관한 내용은 <a href="https://www.kernel.org/doc/html/next/RCU/whatisRCU.html">문서</a>를 참조하시기 바랍니다.<br>
      
      <br><br>
      만약 기존의 Notifier Chain이 다음과 같은 블록으로 이루어져 있다고 가정합니다.<br>
      <span style="{{ site.code }}">... -> A -> B -> C -> ...</span><br>
      <br>
      그리고 새로 추가되는 block을 <span style="{{ site.code }}">D</span> 이고 우선순위는 <span style="{{ site.code }}">A</span> 와 <span style="{{ site.code }}">B</span> 사이 라고 가정하겠습니다.<br>
      <B>*nl</B> 은 <span style="{{ site.code }}">B</span> 를 가리킨 상태에서 우선순위 조건문을 타고 while문을 빠져나옵니다.<br>
      <B>*nl</B> 은 <span style="{{ site.code }}">B</span> 를 가리킵니다.<br>
      <span style="{{ site.code }}">A -> B(*nl) -> C</span>
      <br><br>
      
      이 상태에서 <B>n->next = *nl;</B> 코드로 <span style="{{ site.code }}">D</span> 는 <B>*nl</B> 을 가리키게 됩니다.<br>
      <span style="{{ site.code }}">A -> B(*nl) -> C</span><br>
      <span style="{{ site.code }}">D -> B(*nl) -> C</span><br>
      <br>
      
      <B>rcu_assign_pointer(*nl, n);</B> 코드로 <B>*nl</B> 은 <span style="{{ site.code }}">D</span> 를 가리키게 됩니다.<br>
      <span style="{{ site.code }}">D(*nl) -> B -> C</span>
      <br><br>
      
      따라서, <B>*nl</B> 을 가리키는 <span style="{{ site.code }}">A</span> 는 <span style="{{ site.code }}">D</span> 를 가리키게 됩니다.<br>
      <span style="{{ site.code }}">A -> D(*nl) -> B -> C</span>
      이렇게 <B>Notifier</B> 의 우선순위대로 정렬이 됩니다.<br>

      이렇게 우선순위대로 notifier_block 정리해서 어떻게 사용하냐면,<br>
      함수포인터들을 이렇게 list에 저장했다가 해당 이벤트 발생 시, list에 존재하는 모든 함수들을 호출합니다.<br>
    </p>
  </details>

지금까지 i2cdev 초기화 과정이었습니다.<br>
정리하자면, i2cdev class 디바이스를 만들고,<br>
장치가 버스에 추가/제거 될때마다 attach/detach 함수가 호출 되게끔 notifier를 등록합니다.<br> 
그리고 마지막으로 모듈이 초기화가 될 때 현재 모든 i2c 어댑터를 찾아서 <span style="{{ site.code }}">i2cdev_attach_adapter</span> 함수를 콜합니다.<br>
<br>

그렇다면, i2c 장치의 초기화 패턴에 항상 콜이 되는 <span style="{{ site.code }}">i2cdev_attach_adapter</span> 함수를 보겠습니다.
```
static int i2cdev_attach_adapter(struct device *dev, void *dummy)
{
	struct i2c_adapter *adap;
	struct i2c_dev *i2c_dev;
	int res;
	
	if (dev->type != &i2c_adapter_type)
		return 0;
	adap = to_i2c_adapter(dev);
	
	i2c_dev = get_free_i2c_dev(adap);
	if (IS_ERR(i2c_dev))
		return PTR_ERR(i2c_dev);
	
	cdev_init(&i2c_dev->cdev, &i2cdev_fops);
	i2c_dev->cdev.owner = THIS_MODULE;
	
	device_initialize(&i2c_dev->dev);
	i2c_dev->dev.devt = MKDEV(I2C_MAJOR, adap->nr);
	i2c_dev->dev.class = i2c_dev_class;
	i2c_dev->dev.parent = &adap->dev;
	i2c_dev->dev.release = i2cdev_dev_release;
	dev_set_name(&i2c_dev->dev, "i2c-%d", adap->nr);
	
	res = cdev_device_add(&i2c_dev->cdev, &i2c_dev->dev);
	if (res) {
		put_i2c_dev(i2c_dev, false);
		return res;
	}
	
	pr_debug("i2c-dev: adapter [%s] registered as minor %d\n",
		adap->name, adap->nr);
return 0;
}
```

코드를 하나씩 살펴보면,<br>
<span style="{{ site.code }}">if (dev->type != &i2c_adapter_type)</span> 조건에서 현재 장치가 i2c 장치 타입인지 확인합니다.<br>
그리고 <span style="{{ site.code }}">adap = to_i2c_adapter(dev);</span> 에서 dev를 가지고 있는 구조체의 주소를 adap 변수로 매핑합니다.<br>

이게 무슨 뜻이냐면, <B>to_i2c_adapter</B> 매크로는 다음과 같습니다.<br>
```
#define to_i2c_adapter(d) container_of(d, struct i2c_adapter, dev)
```
<br>

풀어서 보면 다음과 같습니다.
```
struct i2c_adapter {
	...

	device dev;
	int nr;

	...
};
```
```
	if (dev->type != &i2c_adapter_type)
		return 0;
	adap = container_of(dev, struct i2c_adapter, dev);
```
함수의 매개변수 <span style="{{ site.code }}">struct device *dev</span> 의 <B>dev->type</B>이 <B>i2c_adapter_type</B> 이라면,<br>
<span style="{{ site.code }}">dev</span> 는 어느 <span style="{{ site.code }}">struct i2c_adapter</span>변수의 멤버 변수입니다.<br>
<span style="{{ site.code }}">container_of</span> 매크로를 사용해서 매개변수 dev의 값(주소)으로 실제로 dev를 소유하고 있는 구조체 변수(i2c_adapter type)의 주소를 <span style="{{ site.code }}">return</span> 합니다.<br>

첫번째 인자로 들어가는 매개변수 dev 포인터 변수가 가지고 있는 주소 값을 가지고, i2c_adapter 구조체(2번째 param) 와 멤버변수(3번째 param) 사이의 offset 으로 i2c_adapter 구조체의 주소를 포인터변수 <span style="{{ site.code }}">struct i2c_adapter *adap</span> 의 초기값으로 합니다.<br>

<span style="{{ site.code }}">adap->dev</span> 와 매개변수 <span style="{{ site.code }}">dev</span>는 매핑되었습니다.<br>
adap의 type이 동일한 <span style="{{ site.code }}">struct i2c_adapter</span> 이기 때문에 dev를 소유한 변수와 그 변수가 가진 모든 멤버 변수의 주소가 adap의 그것과 동일합니다.<br>
즉, adap을 통해 <span style="{{ site.code }}">struct i2c_adapter</span> 의 dev 외 다른 멤버변수들도 사용할 수 있게 됩니다.<br>

이렇게 하는 이유는 다음과 같습니다.<br>

하드웨어를 초기화를 해야하는데 필요한 정보는 많습니다. 그리고 하드웨어 마다 아키텍처가 다릅니다.<br>
그것을 구조체로 표현을 하기 때문에 하드웨어마다 다 제각각입니다.<br>
하지만, 수 많은 모듈마다 API 포맷이 제각각일 수 없습니다.<br>

그래서 공통적으로 하드웨어와 상호작용을 위한 device 구조체를 멤버변수로 가지고, 각 드라이버 API 매개변수로 dev를 받습니다.<br>
그리고 <span style="{{ site.code }}">container_of</span> 를 통해서 해당 하드웨어 구조체와 동일한 type의 지역변수를 만들고,<br>
매핑을 해서 필요한 정보를 함수 내에 모두 가져다 사용할 수 있게 합니다.<br>

이렇게 해서 매개변수 <B>*dev</B> 를 가지고 i2c_adapter adap 주소를 추적할 수 있게 되었습니다.<br>
이 adap을 가지고 <span style="{{ site.code }}">get_free_i2c_dev</span> 함수를 실행합니다.<br>

```
	i2c_dev = get_free_i2c_dev(adap);
```
```
static struct i2c_dev *get_free_i2c_dev(struct i2c_adapter *adap)
{
	struct i2c_dev *i2c_dev;
	
	if (adap->nr >= I2C_MINORS) {
		printk(KERN_ERR "i2c-dev: Out of device minors (%d)\n",
			adap->nr);
	return ERR_PTR(-ENODEV);
	}
	
	i2c_dev = kzalloc(sizeof(*i2c_dev), GFP_KERNEL);
	if (!i2c_dev)
		return ERR_PTR(-ENOMEM);
	i2c_dev->adap = adap;
	
	spin_lock(&i2c_dev_list_lock);
	list_add_tail(&i2c_dev->list, &i2c_dev_list);
	spin_unlock(&i2c_dev_list_lock);
	return i2c_dev;
}
```

<span style="{{ site.code }}">I2C_MINORS</span> 는 i2c 어댑터가 관리할 수 있는 최대 장치 수를 정의하는 상수 입니다.<br>

i2c 어답터를 매개변수로 받아서, 장치를 초기화 하고 전역 리스트 <span style="{{ site.code }}">i2c_dev_list</span> 에 추가합니다.<br>

그 이후 I2C 디바이스를 초기화 합니다.
```
	cdev_init(&i2c_dev->cdev, &i2cdev_fops);
	i2c_dev->cdev.owner = THIS_MODULE;
	
	device_initialize(&i2c_dev->dev);
	i2c_dev->dev.devt = MKDEV(I2C_MAJOR, adap->nr);
	i2c_dev->dev.class = i2c_dev_class;
	i2c_dev->dev.parent = &adap->dev;
	i2c_dev->dev.release = i2cdev_dev_release;
	dev_set_name(&i2c_dev->dev, "i2c-%d", adap->nr);
	
	res = cdev_device_add(&i2c_dev->cdev, &i2c_dev->dev);
	if (res) {
		put_i2c_dev(i2c_dev, false);
	return res;
	}
	
	pr_debug("i2c-dev: adapter [%s] registered as minor %d\n",
		adap->name, adap->nr);
	return 0;
```

초기화 과정을 보면 알수 있듯이, <span style="{{ site.code }}">container_of</span> 를 사용해<br>
다른 멤버 변수의 값을 사용할 수 있습니다.
```
	i2c_dev->dev.devt = MKDEV(I2C_MAJOR, adap->nr);
```

exit 과정은 init 과정과 반대로 하면 됩니다.<br>
notifier를 unregist 하고, class를 destroy 하고, adapter를 detach 하면 됩니다.
```
static void __exit i2c_dev_exit(void)
{
	bus_unregister_notifier(&i2c_bus_type, &i2cdev_notifier);
	i2c_for_each_dev(NULL, i2cdev_detach_adapter);
	class_destroy(i2c_dev_class);
	unregister_chrdev_region(MKDEV(I2C_MAJOR, 0), I2C_MINORS);
}

static int i2cdev_detach_adapter(struct device *dev, void *dummy)
{
	struct i2c_adapter *adap;
	struct i2c_dev *i2c_dev;
	
	if (dev->type != &i2c_adapter_type)
		return 0;
	adap = to_i2c_adapter(dev);
	
	i2c_dev = i2c_dev_get_by_minor(adap->nr);
	if (!i2c_dev) /* attach_adapter must have failed */
		return 0;
	
	put_i2c_dev(i2c_dev, true);
	
	pr_debug("i2c-dev: adapter [%s] unregistered\n", adap->name);
	return 0;
}
```
