## 소개

ARM은 ARMv8-A 시스템용으로 표준 인터럽트 컨트롤러를 제공합니다.<br>
이 인터럽트 컨트롤러에 대한 프로그래밍 인터페이스는 GIC 아키텍처에 정의되어 있습니다.<br>
GIC 아키텍처 사양은 여러 버전이 있으며, 이 문서는 버전 3( <span style="{{ site.code }}">GICv3</span> )에 집중합니다.<br>
ARM 아키텍처는 <span style="{{ site.code }}">ARMv8-A</span> 기준입니다.<br>

이 포스팅은, 아키텍처에서 정의된 GIC에 대한 내용을 바탕으로<br>
리눅스 커널에서 어떻게 구현 되어있는가가 핵심입니다.<br>

### GIC 개요

ARM은 인터럽트를 다양한 유형으로 정의하고, GIC를 통해 시스템의 복잡한 멀티코어 환경에서<br>
인터럽트를 효과적으로 관리하고 라우팅 합니다.<br>

### 인터럽트 처리 과정

커널 <span style="{{ site.code }}">5.10</span> 을 기준으로 분석합니다.<br>

커널 아키텍처는 인터럽트를 하나의 익셉션으로 처리합니다.<br>
인터럽트는 보안이냐 아니냐에 따라 FIQ / IRQ 로 분류하여 처리합니다.<br>
인터럽트 처리과정은 IRQ를 기준으로 따라가겠씁니다.(리눅스 커널은 FIQ를 취급하지 않음)<br>

ARM은 익셉션이 발생하면, 현재 프로그램 상태( <span style="{{ site.code }}">PSTATE</span> ) 와 발생한 지점을 백업하고,<br>
<span style="{{ site.code }}">PSTATE</span> 를 익셉션에 맞게 업데이트 합니다.<br>
그리고 프로그램 카운터를 익셉션 백터로 분기합니다.<br>
위 과정이 하드웨어로 동작하기 때문에 소프트웨어 코드를 찾을 수 없습니다.<br>
코드 분석이 가능한 부분은 이 과정(PSTATE, ELR_ELx, ESR_ELx 백업, 업데이트, PC값 익셉션 벡터로 업데이트) 이후 동작입니다.<br>

다음은 arch/arm64/kernel/entry.S 의 코드 중 일부 입니다.
```
SYM_CODE_START(vectors)
        kernel_ventry   1, sync_invalid                 // Synchronous EL1t
        kernel_ventry   1, irq_invalid                  // IRQ EL1t
        kernel_ventry   1, fiq_invalid                  // FIQ EL1t
        kernel_ventry   1, error_invalid                // Error EL1t

        kernel_ventry   1, sync                         // Synchronous EL1h
        kernel_ventry   1, irq                          // IRQ EL1h
        kernel_ventry   1, fiq_invalid                  // FIQ EL1h
        kernel_ventry   1, error                        // Error EL1h

        kernel_ventry   0, sync                         // Synchronous 64-bit EL0
        kernel_ventry   0, irq                          // IRQ 64-bit EL0
        kernel_ventry   0, fiq_invalid                  // FIQ 64-bit EL0
        kernel_ventry   0, error                        // Error 64-bit EL0

#ifdef CONFIG_COMPAT
        kernel_ventry   0, sync_compat, 32              // Synchronous 32-bit EL0
        kernel_ventry   0, irq_compat, 32               // IRQ 32-bit EL0
        kernel_ventry   0, fiq_invalid_compat, 32       // FIQ 32-bit EL0
        kernel_ventry   0, error_compat, 32             // Error 32-bit EL0
#else
        kernel_ventry   0, sync_invalid, 32             // Synchronous 32-bit EL0

        kernel_ventry   0, fiq_invalid, 32              // FIQ 32-bit EL0
        kernel_ventry   0, error_invalid, 32            // Error 32-bit EL0
#endif
SYM_CODE_END(vectors)
```
