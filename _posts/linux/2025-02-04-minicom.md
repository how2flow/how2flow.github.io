## minicom

Minicom is a terminal program for Linux and Unix-like systems.<br>
I mainly use it with a [USB-UART module](https://www.hardkernel.com/shop/usb-uart-2-module-kit-copy/).<br>

### Installing minicom and basic setup

Install the apt package and set the basic options.
```
$ sudo apt-get install minicom
```
<br>

Run minicom.
```
$ sudo minicom
minicom: cannot open /dev/modem: No such file or directory
```
<br>

When running it, you may encounter the following error log.<br>
<span style="{{ site.code }}">minicom: cannot open /dev/modem: No such file or directory</span><br>

If you google it, it says using the <span style="{{ site.code }}">-s</span> option solves it.
```
$ sudo minicom --help
Usage: minicom [OPTION]... [configuration]
A terminal program for Linux and other unix-like systems.

  -b, --baudrate         : set baudrate (ignore the value from config)
  -D, --device           : set device name (ignore the value from config)
  -s, --setup            : enter setup mode
  -o, --noinit           : do not initialize modem & lockfiles at startup
  -m, --metakey          : use meta or alt key for commands
  -M, --metakey8         : use 8bit meta key for commands
  -l, --ansi             : literal; assume screen uses non IBM-PC character set
  -L, --iso              : don't assume screen uses ISO8859
  -w, --wrap             : Linewrap on
  -H, --displayhex       : display output in hex
  -z, --statline         : try to use terminal's status line
  -7, --7bit             : force 7bit mode
  -8, --8bit             : force 8bit mode
  -c, --color=on/off     : ANSI style color usage on or off
  -a, --attrib=on/off    : use reverse or highlight attributes on or off
  -t, --term=TERM        : override TERM environment variable
  -S, --script=SCRIPT    : run SCRIPT at startup
  -d, --dial=ENTRY       : dial ENTRY from the dialing directory
  -p, --ptty=TTYP        : connect to pseudo terminal
  -C, --capturefile=FILE : start capturing to FILE
  -F, --statlinefmt      : format of status line
  -R, --remotecharset    : character set of communication partner
  -v, --version          : output version information and exit
  -h, --help             : show help
  configuration          : configuration file to use

These options can also be specified in the MINICOM environment variable.
This variable is currently unset.
The configuration directory for the access file and the configurations
is compiled to /etc/minicom.

Report bugs to <minicom-devel@lists.alioth.debian.org>.
```
You can see that the <span style="{{ site.code }}">-s</span> option runs minicom in setup mode.<br>
Besides <span style="{{ site.code }}">-s</span>, the <span style="{{ site.code }}">-D</span> and <span style="{{ site.code }}">-b</span> options are also frequently used.<br>
<br>

Run it in setup mode.
```
$ sudo minicom -s
```
<br>

Here you can configure basic settings.<br>
Since most can be changed with options anyway, I only changed a few simple things.<br>
<br>

<p align="center">
  <img src="/assets/images/posts/contents/linux/minicom_flow.png" alt="minicom_flow" width="640" height="480">
</p>
In <span style="{{ site.code }}">Serial port setup</span>, disable both H/W and S/W Flow control, and<br>
<br>

<p align="center">
  <img src="/assets/images/posts/contents/linux/minicom_key.png" alt="minicom_key" width="640" height="480">
</p>
In <span style="{{ site.code }}">Screen and keyboard</span>, I changed the command key to <span style="{{ site.code }}">Ctrl + M</span>.<br>
<br>

Flow-Control is used for synchronous communication only when the buffer in UART is at maximum. A representative example is <span style="{{ site.code }}">RS232</span>.<br>
H/W flow-control is only allowed when using cts and rts pins.<br>
S/W flow-control means it will be implemented via a program.<br>
Since neither applies, set them all to No.<br>
<br>
You can use any command key you prefer.<br>
I changed it because tmux's prefix overlaps with minicom's default command key.<br>
Now, save the settings with <span style="{{ site.code }}">Save setup as dfl</span> and exit.<br>

What you need when using <span style="{{ site.code }}">minicom</span> is the connected device and the bitrate.<br>
The connected device is set to <span style="{{ site.code }}">/dev/modem</span> by default, but<br>
looking at the help message, you can set the device with the <span style="{{ site.code }}">-D</span> option.<br>

### Running minicom

I used a USB-UART module.<br>
By plugging and unplugging the device from the desktop hub, I check which device node it is.<br>
For me, <span style="{{ site.code }}">/dev/ttyUSB0</span> appears and disappears.<br>

Once you find the device to connect to and decide the desired bitrate, open a terminal using minicom.<br>
Let's assume the bitrate is 115200.<br>

device: /dev/ttyUSB0<br>
bitrate: 115200<br>

```
$ sudo minicom -D /dev/ttyUSB0 -b 115200
```

You can also add the <span style="{{ site.code }}">-w</span> option when running minicom.<br>

### Summary

1. <span style="{{ site.code }}">sudo apt-get install minicom</span><br>
2. <span style="{{ site.code }}">sudo minicom -s</span><br>
3. Set all <span style="{{ site.code }}">Flow-Control</span> to <span style="{{ site.code }}">No</span> and change the command key. (<span style="{{ site.code }}">Ctrl + M</span>)<br>
4. Connect to the device using <span style="{{ site.code }}">minicom</span> with the <span style="{{ site.code }}">-D</span> and <span style="{{ site.code }}">-b</span> options.<br>
5. Turn <span style="{{ site.code }}">LineWrap</span> <span style="{{ site.code }}">ON</span> using the <span style="{{ site.code }}">Command key + Z</span> combination, or add the <span style="{{ site.code }}">-w</span> option when connecting the device.<br>
