Most devices use DHCP for IP address assignment.<br>
It's convenient because IP addresses are assigned automatically, requiring little effort from network administrators.<br>
However, during development or testing, having the IP address change constantly can be quite troublesome.<br>

This creates a need for either a temporary or permanent static IP.<br>
Here's how to assign a static IP in an Ubuntu environment.<br>

## Assigning a Static IP Address

Ubuntu stores network-related scripts in the <span style="{{ site.code }}">/etc/netplan/</span> directory within the Linux filesystem.<br>
In Ubuntu 22.04, if you check <span style="{{ site.code }}">/etc/netplan/</span>,<br><br>

you will find the <span style="{{ site.code }}">01-network-manager-all.yaml</span> file.<br>
A YAML file is essentially a data set file in the format <span style="{{ site.code }}">label: data</span>.<br>

Let's look at the file content:
```
# Let NetworkManager manage all devices on this system
network:
  version: 2
  renderer: NetworkManager
```

It's written in <span style="{{ site.code }}">label: data</span> format,<br>
with sub-properties distinguished by indentation (indent=2).<br>

A <span style="{{ site.code }}">renderer</span> attribute value of <span style="{{ site.code }}">NetworkManager</span>, as seen above, is used for desktop images with GUI packages installed.<br>

For standard server images without a GUI, you can use <span style="{{ site.code }}">networkd</span>.<br><br>

Based on my desktop, the network ID is <span style="{{ site.code }}">192.168.11</span>,<br>
so I won't change the <span style="{{ site.code }}">192.168.11.</span> portion.<br>


First, identify the network device name.
```
$ ifconfig
```

  <img src="/assets/images/posts/contents/linux/net-dev-list.png" alt="net-dev-list" width="640" height="480"><br>
  <span style="{{ site.img }}">[picture 1] ifconfig network list</span>
</p>
<br>

The Ethernet device name is <span style="{{ site.code }}">enp2s0</span>, and the Wi-Fi device name is <span style="{{ site.code }}">wlp1s0</span>.<br>

### Ethernet

This is the modified version to assign a static IP address of <span style="{{ site.code }}">192.168.11.100</span> instead of <span style="{{ site.code }}">192.168.11.???</span>
```
# Let NetworkManager manage all devices on this system
network:
  version: 2
  renderer: NetworkManager
  ethernets:
    enp2s0:
      dhcp4: no 
      addresses: [192.168.11.100/24]
      gateway4: 192.168.11.1
```

### Wi-Fi

This is the modified version to assign a static IP address of <span style="{{ site.code }}">192.168.11.100</span> instead of <span style="{{ site.code }}">192.168.11.???</span>
```
# Let NetworkManager manage all devices on this system
network:
  version: 2
  renderer: NetworkManager
  wifis:
    wlp1s0:
      optional: true
      access-points:
        "SSID-NAME":
          password: "AP's password"
      dhcp4: no 
      addresses: [192.168.11.100/24]
      gateway4: 192.168.11.1
```


## Applying the Changes

If you lose internet access, simply change the <span style="{{ site.code }}">dhcp4</span> attribute value back to yes.<br>
(Note: You might see both the static and randomly assigned addresses simultaneously.)<br><br>

You can either reboot the system or use the command <span style="{{ site.code }}">sudo netplan apply</span>.<br>
