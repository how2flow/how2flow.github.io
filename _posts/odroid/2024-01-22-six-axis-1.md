I've brought an interesting topic today.<br>
I was thinking about making something fun with an Odroid and decided to pick up a robot arm I had at home.<br>

You can purchase it at [link](aliexpress.com/item/1005005605170804.html?pdp_npi=2%40dis%21KRW%21₩38%2C245%21₩21%2C031%21%21%21%21%21%4021015b7d16883512636385872e921f%2112000033723251352%21btf&_t=pvid%3Aec9cc8c5-34a0-43c6-8cc1-d0a8d3d1d15c&afTraceInfo=1005005605170804__pc__pcBridgePPC__xxxxxx__1688351264&spm=a2g0o.ppclist.product.mainProduct&gatewayAdapt=glo2kor) or from local retailers.<br>
The motor driver ( <span style="{{ site.code }}">pca9685</span> ) and motors (6x <span style="{{ site.code }}">MG996R</span> ) must be purchased separately.<br>

While control can be implemented in C or Python, I chose ROS2 as I wanted to gain experience with it.<br>
I'll take my time to add features one by one and complete the project.<br>

The requirements are as follows:
```
- Odroid C4       ... (x1)
- MG996R          ... (x6)
- PCA9685         ... (x1)
- Robot frame set ... (x1)
```

You need to set up the environment to use the pca9685 on the C4 board in advance.<br>
Refer to this [post](/posts/pca9685/) and the [package documentation](https://github.com/how2flow/ros2_axis6#fixups) for the setup.<br>
For the OS, use the [C4 ROS2 image](https://dn.odroid.com/S905X3/ODROID-C4/ROS2/ubuntu-20.04-ros2-odroidc4-20230105.img.xz).<br>

```
$ git clone --recurse-submodules https://github.com/how2flow/ros2_axis6
```
<br>

Simply downloading the source code won't make the programs run immediately.<br>
As mentioned above, configure the ROS2 package execution environment referring to the [README](https://github.com/how2flow/ros2_axis6/blob/master/README.md).<br>

You can leave the breadboard as shown in the photo.<br>

Connect an external power supply (at least 6V/3A) to the pca9685.<br>
Build the downloaded ROS2 package and run the program.
```
$ cd ~/robot_ws
$ ln -s ~/ros2_axis6 .
$ ln -s ~/ros2_axis6/how2flow_interfaces .
$ cb
$ source ~/.bashrc
$ ros2 run axis6 operator
```
