## Introduction
<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_only.jpg" alt="multiio_only" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 1] MULTIIO</span>
</p>
<br>

This board is designed to interface with the ODROID-M1S.<br>
It was created to improve the process of selecting interfaces in <span style="{{ site.code }}">/boot/config.ini</span> and identifying pin locations.<br>
It allows for simultaneous use of all interfaces, with each interface clearly labeled on the MULTIIO board.<br>

## Using MULTIIO

Below are the usage instructions and descriptions for MULTIIO.<br>

### Getting Started with MULTIIO

First, prepare the ODROID-M1S and MULTIIO boards.<br>
Connect the MULTIIO board to the M1S while the power is off.<br>
<p align="center">
  <img src="/assets/images/posts/contents/odroid/m1s_and_multiio.jpg" alt="multiio_only" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 2] m1s and multiio</span>
</p>
<br>

Download the Ubuntu image for the M1S from the official site,<br>
flash it to the M1S eMMC, and boot the system.<br>

Modify <span style="{{ site.code }}">/boot/config.ini</span> on the M1S as follows:
```
[generic]
#default_console=ttyFIQ0
overlay_resize=16384
overlay_profile=
overlays="board_multiio"
 
[overlay_custom]
overlays="i2c0 i2c1"
 
[overlay_hktft32]
overlays="hktft32 ads7846"
```
<br>

<span style="{{ site.code }}">reboot</span> the system.<br>

### MULTIIO Interfaces

The MULTIIO board provides various interfaces including <span style="{{ site.code }}">canfd</span>, <span style="{{ site.code }}">gpio</span>, and <span style="{{ site.code }}">i2c</span>.<br>

#### gpio

<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_gpio.jpg" alt="multiio_gpio" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 3] multiio gpio</span>
</p>
<br>

You can easily test sensors or devices controllable via GPIO.<br>
The sysfs number is 14, and it can be controlled with <span style="{{ site.code }}">root</span> privileges.
```
$ echo 14 | sudo tee /sys/class/gpio/export
```

INPUT
```
$ echo in | sudo tee /sys/class/gpio/gpio14/direction
$ cat /sys/class/gpio/gpio14/value
```

OUTPUT
```
$ echo out | sudo tee /sys/class/gpio/gpio14/direction
$ echo [1|0] | sudo tee /sys/class/gpio/gpio14/value
```

#### i2c

I2C is straightforward, and additional examples are available on the [Wiki](https://wiki.odroid.com/accessory/add-on_boards/multiioboard#weather-board_zero).<br>
I recommend referring to the Wiki.<br>

#### rtc

<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_rtc.jpg" alt="multiio_rtc" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 5] multiio rtc</span>
</p>
<br>

While the M1S has a built-in RTC, MULTIIO also supports one.<br>
To use the MULTIIO RTC, you must switch from the default system RTC to the MULTIIO RTC.<br>
**The coin battery is sold separately.**<br>

Among various methods, I'll use <span style="{{ site.code }}">systemd</span> to select and automatically initialize the RTC during boot.<br>

First, create the udev rules.
```
$ echo "KERNEL==\"rtc1\", TAG+=\"systemd\"" | sudo tee /etc/udev/rules.d/50-rtc.rules
```
<br>

Define the service.
```
$ sudo vi /lib/systemd/system/hwrtc.service
```
```
[Unit]
Description=Synchronise System clock to hardware RTC
DefaultDependencies=no
Wants=dev-rtc1.device
After=dev-rtc1.device
Before=systemd-journald.service time-sync.target sysinit.target shutdown.target
Conflicts=shutdown.target
 
[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/sbin/hwclock --hctosys --rtc=/dev/rtc1 -v --utc --noadjfile
RestrictRealtime=yes
 
[Install]
WantedBy=sysinit.target
```
<br>

In the Unit section, <span style="{{ site.code }}">dev-rtc1.device</span> refers to the device node <span style="{{ site.code }}">/dev/rtc1</span>.<br>

Enable the service.
```
$ sudo systemctl enable hwrtc
$ sudo reboot
```
<br>

After rebooting, check the service status.

#### rs232

<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_rs232.jpg" alt="multiio_rs232" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 8] multiio rs232</span>
</p>
<br>

It can be connected to a computer using a <span style="{{ site.code }}">USB-to-Serial Cable</span>.<br>
<span style="{{ site.code }}">hardware flow control</span> is also supported.<br>
You can test <span style="{{ site.code }}">hardware flow control</span> by connecting and disconnecting the cts-rts lines.<br><br>

Check the device node.
```
$ ls /dev/ttyS?
/dev/ttyS0
```
<br>

Commands for RS232 operation:
```
$ sudo stty -F /dev/ttyS0 9600 -cstopb -parenb

# receive
$ cat /dev/ttyS0

# transmit
$ echo "123" | sudo tee /dev/ttyS0
```
<br>

#### canfd

<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_canfd.jpg" alt="multiio_canfd" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 9] multiio canfd</span>
</p>
<br>

As you can see, an additional canfd module is required.<br>
To use CAN-FD with MULTIIO, modify <span style="{{ site.code }}">/boot/config.ini</span> as follows:
```
[generic]
#default_console=ttyFIQ0
overlay_resize=16384
overlay_profile=
overlays="board_multiio board_multiio_canfd"
 
[overlay_custom]
overlays="i2c0 i2c1"
 
[overlay_hktft32]
overlays="hktft32 ads7846"
```
<br>

Check the CAN device.
```
$ lsmod
mcp251xfd              32768  0
can_dev                24576  1 mcp251xfd
```

Keen observers might notice that the M1S has native CAN-FD.<br>
MULTIIO uses a CAN-FD module with an on-board <span style="{{ site.code }}">mcp2517</span> chip.<br>

The first reason is that certain issues occurred during the development phase.<br>
Secondly, I wanted to ensure compatibility with CAN-FD modules.<br>
Thirdly, the pin layout offers the advantage of supporting up to two CAN channels.<br>
However, once issues are resolved or if the dual-channel advantage is deemed unnecessary, the CAN-FD module might be replaced with native CAN in future releases to enhance price competitiveness through cost reduction.<br>

## History (Pre-release Development Phase)

Although MULTIIO seems simple, debugging and testing it was not easy.<br>
There were several releases prior to the final version.<br>

<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_20230719.jpg" alt="multiio_20230719" width="480" height="320"><br>
  <span style="{{ site.img }}">[Picture 11-1] multiio 20230719</span>
</p>
<br>

It looks quite different from the current <span style="{{ site.code }}">20230810</span> version.<br>
The differences are obvious when compared side-by-side.<br>

<p align="center">
  <img src="/assets/images/posts/contents/odroid/multiio_compare.jpg" alt="multiio_compare" width="480" height="320"><br>

The RTC coin battery became smaller, and CAN-FD was changed to an option using the mcp2517.<br>
A lot of attention was paid to details during debugging.<br>
