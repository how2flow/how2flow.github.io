## Introduction

Trace32 is a powerful debugging tool.<br>
It requires a separate license and connects via a debug interface, most commonly <span style="{{ site.code }}">JTAG</span>.<br>
This is a personal reference post summarizing basic commands.<br>
It will be updated continuously on the [Trace32 Quick Start Guide](/trace32/trace32/) page.<br>
For OS awareness, please refer to the [Trace32 OS Awareness](/trace32/t32-os-awareness/) page.<br>

## Getting Started

To use Trace32 (abbreviated as T32), you must first install the debugging software.<br>
However, due to the high cost of licenses, setup in a general environment is challenging.<br>
This guide assumes that the basic software and license are already prepared.<br>

### Device Connection

Connect the target board to T32.<br>
Ensure the target board is powered off to protect both the debugger and the board.<br>
Carefully identify pin 1 of the debug interface (usually JTAG) and make the connection.<br>

You can monitor the debugger's status via the Power Debugger interface LEDs:
```
' POWER
' SELECT
' EMULATE
```
<span style="{{ site.code }}">POWER</span> : Lights up when the power is on.<br>
<span style="{{ site.code }}">SELECT</span> : Flashes on power-up; stays on while PowerView is running.<br>
<span style="{{ site.code }}">EMULATE</span> : Lights up when the target is running.<br>

Start by running <span style="{{ site.code }}">t32m[core].exe</span> or <span style="{{ site.code }}">t32start.exe</span> to select the core.<br>

### PowerView

Option 1: Configure target information.<br>
  Go to [CPU] -> [System Setting] -> ... and configure JTAG clock, multicore, and other settings as needed, then select "UP" in the Mode menu.<br>
  <br>

Option 2: Use a pre-written script for the target board.<br>
  Load a .cmm script via [File] -> [Edit script].<br>
  Execute using <span style="{{ site.code }}">Do</span> or <span style="{{ site.code }}">Debug</span> -> <span style="{{ site.code }}">over</span>.<br>
  Scripts can also be loaded via the command line.<br>
  You can use <span style="{{ site.code }}">pedit {cmm script}</span> or <span style="{{ site.code }}">pedit * </span>.<br>
  As mentioned later, using <span style="{{ site.code }}">pedit * </span> opens a file browser.<br>
  <br>

Option 3: Be cautious when using T32 in environments with Watchdogs (e.g., Linux kernel).<br>
  If the system hits a breakpoint in T32, the Watchdog might automatically reset the device to prevent it from hanging.<br>
  This will cause the connection with T32 to drop.<br>

  Refer to the datasheet to disable the WDT, or for Linux, remove the relevant CONFIG and rebuild the kernel.<br>

  Additionally, in multicore environments, there might be mechanisms monitoring the IDLE states between different cores.<br>
  In this case, set the CPU configuration to MPCore and include information for each core in the T32 script.<br>
  <br>

## Commands

Enter commands at the command prompt (B::).<br>
Commands are case-insensitive, though they may vary between software versions.<br>
Use the Tab key for auto-completion to find the correct command if there are differences.<br>

### Basic

<B>reset</B>
```
B:: sys.d
B:: reset
```
<br>

<B>Go</B>
```
B:: g
```
Enters the running state. Register dumps are not available during this time.<br>
<br>

<B>Break</B>
```
B:: b
```
Enters the break state. You can inspect register values in this state.<br>
<br>

<B>Data</B>
```
B:: Data.{Set/dump/...} {class}:{address} /{option} ..
```
This command is used to check register values at specific addresses.<br>
The 'class' determines the address attributes.<br>
For example, the following classes exist:
```
a: Represents a physical address.
d: Recognizes as a data type.
s: Secure access.
ns: Non-secure access.
spr: Accesses via architecture-encoded addresses.
```
<br>

<B>Load elf</B>
```
B:: Data.LOAD.ELF {target_elf_path} /{option}
```
This command loads symbols.<br>
You can use '\*' instead of target_elf_path to manually select a file.<br>
After execution, view the symbols via [View] -> [Symbol Table] in PowerView.<br>
Similar to cmm scripts or symbol files, using <span style="{{ site.code }}"> * </span> when loading files opens a file explorer.<br>
You can then select the file directly using the explorer.<br>
<br>

<B>Translate Path</B>
```
B:: sYmbol.sourcePATH.translate "{invalid part}" "{correct part}"
```
The source paths in the loaded file might sometimes be unknown to the local system.<br>
Use this to replace an invalid path segment with the correct local path.<br>
<br>

## CMM Scripts

Use the following command to write a cmm script:
```
B:: pedit main.cmm
```
<br>

Or to import an existing script (using file explorer):
```
B:: pedit *
```
<br>

Some script examples (for reference)
```
/* data dump */
data.dump R(PC) // data dump in PC addr
data.dump apb:0x... // access memory addr directly
data.dump ahb:0x... // access memory addr directly
data.dump a:0x... // access memory addr directly
data.dump a:0x... /quad // access memory addr directly

/* mmu */
mmu.list.pt R(x3) // check virtual_addr to physical_addr
```
